# TRABAJO-EDA---Sistema de Gestión de Trámite Documentario

Este proyecto es una aplicación desarrollada en **Java** que permite gestionar y organizar solicitudes documentarias utilizando estructuras de datos fundamentales desarrolladas manualmente.

## Propósito del Proyecto
Optimizar el seguimiento de expedientes y la priorización de atención mediante un sistema eficiente que reemplaza los procesos manuales, permitiendo un control total sobre el estado de cada trámite.

## Estructuras de Datos Implementadas (TADs)
A diferencia de usar librerías estándar, para este proyecto se desarrollaron desde cero:
* **Listas Simples y Dobles**: Para la gestión de registros.
* **Listas Circulares**: Para procesos de revisión continua.
* **Pilas (Stacks)**: Para el historial de estados de los trámites.
* **Colas (Queues)**: Para la atención por orden de llegada y prioridad.


## Objetivos Destacados
* **Priorización**: Alertas automáticas según antigüedad.
* **Trazabilidad**: Seguimiento real entre dependencias universitarias.
* **Interfaz Gráfica**: GUI intuitiva desarrollada en Java Swing/AWT.

## Metodología
Desarrollo **Incremental** bajo el paradigma de **Programación Orientada a Objetos (POO)**.
